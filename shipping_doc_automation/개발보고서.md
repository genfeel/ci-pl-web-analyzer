# 개발보고서 — 케이스 생성 사유(reason) 추적 기능

- 작성일: 2026-03-10
- 목적: CI→PL 변환 시 각 케이스의 포장 사유/근거를 CLI verbose 출력에 표시

---

## 1. 개요

기존에는 CI→PL 변환 결과에서 **왜 해당 방식으로 포장되었는지** 근거가 표시되지 않았음.
이번 변경으로 각 Packer가 케이스 생성 시 `reason` 문자열을 설정하고,
CLI verbose 모드(`-v`)에서 케이스별 사유를 확인할 수 있도록 개선함.

- 엑셀 PL 출력에는 변경 없음 (CLI verbose 전용)

---

## 2. 수정 파일 및 변경 내용 (6개 파일)

### 2-1. `src/models/data_models.py` — PackedCase에 reason 필드 추가

**변경 위치:** Line 107

`PackedCase` dataclass에 `reason: str = ""` 필드를 추가하여, 모든 케이스가 생성 사유를 보유할 수 있도록 함.

```python
@dataclass
class PackedCase:
    """하나의 포장 케이스"""
    case_no: int
    case_type: str
    items: list = field(default_factory=list)
    dimensions: tuple = (0, 0, 0)
    case_weight: float = 0.0
    category: ProductCategory = ProductCategory.UNKNOWN
    category_header: str = ""
    order_number: str = ""
    reason: str = ""          # ← 신규 추가
```

### 2-2. `src/packing/vcb_packer.py` — VCB 개별포장 사유

**변경 위치:** Line 37~51

PackedCase 생성 직전에 reason 문자열을 조합하여 전달.

```python
reason = (
    f"VCB 개별포장: 1대/케이스 규칙 "
    f"({item.model_number}, 단중 {unit_weight:.1f}kg)"
)

case = PackedCase(
    ...,
    reason=reason,
)
```

**출력 예시:**
```
VCB 개별포장: 1대/케이스 규칙 (HGV3141, 단중 70.0kg)
```

## 5. 영향 범위

- **CLI verbose 출력 (`-v`):** 케이스별 사유 줄 추가 표시
- **엑셀 PL 출력:** 변경 없음
- **기존 기능:** `reason` 필드 기본값이 `""`이므로 기존 코드에 영향 없음
- **하위 호환성:** PackedCase를 사용하는 다른 모듈에서 reason을 참조하지 않으면 동작 변화 없음
