1. 프로젝트 개요

1.1 배경

HD현대일렉트릭 배전전략영업과에서 월 200건에 달하는 선적서류를 담당자 2인이 수작업으로 작성 중이며, 개인 경험에 의존하는 PL(Packing List) 작성의 자동화가 필요하다.

1.2 목표

CI(Commercial Invoice) 엑셀 파일을 입력받아 PL(Packing List)을 자동 생성하는 Python CLI 프로토타입을 개발한다.

1.3 접근 방식

규칙 기반 알고리즘 + ML 보정 (하이브리드)

CI 엑셀 파싱 → 제품 분류 → 포장 전략 선택 → 적재 최적화 → 중량 산정 → PL 엑셀 출력

4.4 제품 분류기 (src/classifier/product_classifier.py)

3단계 분류 전략 (우선순위 순):

0단계: SPARE 헤더 검사 → "SPARE PART FOR..." 헤더면 즉시 SPARE
1단계: 카테고리 헤더 기반 → CI 문서 구조에서 가장 신뢰도 높음
2단계: 모델번호 접두사 기반 → 헤더 없는 경우 폴백
3단계: Description 텍스트 → 최종 폴백

  • A-FRAME (800A 이하) → 케이스당 최대 6대
  • B-FRAME (1600A 이하) → 케이스당 최대 4대
  • C-FRAME (2000A 이상) → 케이스당 최대 2대

5. ML 중량 예측 모듈

5.1 개요

미등록 제품의 단위중량을 예측하기 위한 보조 ML 모듈이다. 주 중량 결정은 product_db.json 룩업 + 규칙 기반 추정이며, ML은 두 방법 모두 실패할 때의 최종 폴백으로 동작한다.

5.2 알고리즘: GradientBoostingRegressor

핵심 원리: 약한 학습기(결정 트리)를 순차적으로 학습시켜, 각 트리가 이전 트리들의 잔차(오차)를 보정하는 앙상블 기법이다.

F₀(x) = mean(y)                          # 초기: 전체 평균
F₁(x) = F₀(x) + η × tree₁(residual₁)    # 1차 보정
F₂(x) = F₁(x) + η × tree₂(residual₂)    # 2차 보정
...
Fₙ(x) = Fₙ₋₁(x) + η × treeₙ(residualₙ) # n차 보정

여기서 residualₜ = y - Fₜ₋₁(x)는 손실함수의 음의 기울기(negative gradient)이며, 이것이 "Gradient" Boosting이라는 이름의 유래이다.

5.3 피처 설계

모델번호에서 5개 수치 특성을 추출한다:

피처 설명 예시 (HGM630)
family_id 제품 계열 ID 6 (HGM)
log_current log(전류정격+1) log(631) ≈ 6.45
frame_type 프레임 타입 인코딩 0
frame_size 프레임 크기 인코딩 4 (630AF)
log_base_weight log(계열 기본중량) log(4) ≈ 1.39

5.4 과적합 분석 및 해결

초기 구현에서 심각한 과적합이 발견되어 수정하였다.

원인 분석

요인 상세
데이터 부족 60개 샘플, 피처 7개 (비율 8.6:1, 권장 50:1 이상)
타겟 편향 0.14kg~200kg, 1,429배 범위
과도한 복잡도 100 트리 × depth 4 → 수천 리프노드가 60개에 맞춤
피처 중복 current_rating = model_numeric (동일값)

지표 수정 전 수정 후

Train MAE 4.9 kg 12.3 kg
CV MAE 23.1 kg 12.0 kg
Train-CV MAE 갭 18.2 kg 0.3 kg

적용한 과적합 방지 전략

전략 변경 내용 효과

5.7 학습 데이터 확장 전략

NNLS(Non-Negative Least Squares)를 사용하여 기존 PL에서 단위중량을 역산한다:

케이스 순중량 = Σ(모델별 수량 × 단위중량)

→ 연립방정식 A·x = b 구성
  A[i,j] = 케이스 i에서 모델 j의 수량
  b[i]   = 케이스 i의 총 순중량
  x[j]   = 모델 j의 단위중량 (미지수)

→ scipy.optimize.nnls로 비음수 해 도출
